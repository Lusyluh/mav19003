Client-side Validation

Client-side validation is an initial check and an important feature of good user experience; by catching invalid data on the client-side, the user can fix it straight away. If it gets to the server and is then rejected, a noticeable delay is caused by a round trip to the server and then back to the client-side to tell the user to fix their data.

What is form validation?

When entering data, the browser and/or the web server usually check to see that the data is in the correct format and within the constraints set by the application. Validation done in the browser is called client-side validation, while validation done on the server is called server-sidevalidation. In this chapter we are focusing on client-side validation.

Reasons to validate:

  • to get the right data, in the right format
  • to protect our users' data. Forcing our users to enter secure passwords makes it easier to protect their account information.
  • to protect ourselves. There are many ways that malicious users can misuse unprotected forms to damage the application

Using built-in form validation

  • required: Specifies whether a form field needs to be filled in before the form can be submitted.
  • minlength and maxlength: Specifies the minimum and maximum length of textual data (strings)
  • min and max: Specifies the minimum and maximum values of numerical input types
  • type: Specifies whether the data needs to be a number, an email address, or some other specific preset type.
  • pattern: Specifies a regular expression that defines a pattern the entered data needs to follow.
